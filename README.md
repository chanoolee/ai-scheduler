💡 1. 기획 의도 (Background & Purpose)
병원 등 교대 근무 환경에서는 '월별 만근 시간', '연속 근무 제한', '근무자별 오프(Off) 요청' 등 까다로운 제약 조건이 거미줄처럼 얽혀 있습니다. 매월 수동으로 시간표를 짜는 데 막대한 리소스가 소모되는 현장의 페인포인트(Pain-point)를 해결하고자 본 프로젝트를 기획했습니다.
100% AI에 의존하는 맹목적인 자동화가 아니라, 관리자의 '수동 개입'과 AI의 '자동 완성'이 완벽하게 조화를 이루는 하이브리드 스케줄링 아키텍처를 고안하여 실무 사용성을 극대화하는 데 초점을 맞추었습니다.

🏗️ 2. 핵심 시스템 설계 및 제약 조건 제어 (System Design)
본 프로젝트의 핵심은 화면 UI부터 백엔드 상태 관리까지, 복잡한 비즈니스 룰을 유연하게 제어하는 '설계'에 있습니다.

1. 사용자 맞춤형 근무 타입(Shift Type) 추상화
 - 고정된 근무 틀이 아닌, 각 병원(사용자 ID)의 특성에 맞춰 근무 타입과 병원 운영 시간을 자유롭게 커스터마이징 할 수 있도록 데이터 구조를 유연하게 설계했습니다.
2. 하이브리드 상태(State) 동기화 파이프라인
 - 사용자가 캘린더 화면에 수동으로 먼저 확정해 둔 근무 시간이나 휴무일 데이터를 AI가 완벽하게 인지하고, 나머지 빈 공간만 제약 조건에 맞춰 최적화하여 채워 넣도록 상태 관리 로직을 세분화했습니다.
3. AI 자동화 '고정 조건' 주입 인터페이스
 - AI가 스케줄을 자동 생성할 때 절대 어기면 안 되는 '인당 월별 만근 시간' 등의 핵심 제약 조건을 텍스트 박스 형태로 입력받아 프롬프트에 동적으로 맵핑되도록 설계했습니다.
4. UX/UI 렌더링 최적화
 - 초기 매트릭스 형태의 시간표가 데이터 증가에 따라 가로로 길어져 가독성이 떨어지는 문제를 선제적으로 파악했습니다. 이에 따라 상단에는 직관적인 시간표를, 하단에는 AI 제어 텍스트 박스를 배치하는 수직(Vertical) 배열로 화면 구성을 리팩토링하여 사용자 경험을 개선했습니다.

🤖 3. Vibe Coding 도입 및 개발 방법론
최신 AI 코딩 트렌드인 바이브코딩(Vibe Coding) 방식을 적극 활용하여 프로젝트를 진행했습니다.
이는 단순히 AI가 코드를 짜주게 방치한 것이 아닙니다. 개발자인 저는 전체 시스템의 뼈대(Architecture)를 잡고, 복잡한 교대 근무 알고리즘의 엣지 케이스를 정의하는 데 역량을 집중했습니다. 이후 구체적인 기능 구현과 프론트엔드/백엔드 보일러플레이트 작성은 AI 툴을 활용해 개발 생산성을 압도적으로 끌어올리는 모던한 개발 방법론을 실증했습니다.

🛠️ 4. 기술 스택 (Tech Stack)
비즈니스 로직 처리에 최적화된 모던 아키텍처 기반으로 큰 틀을 구성했습니다.
 - Backend: Python (FastAPI) - 비동기 처리 및 AI 모듈 연동에 최적화된 경량 프레임워크
 - Frontend: React - 상태 변화가 잦은 복잡한 시간표 컴포넌트의 렌더링 최적화
 - Database: PostgreSQL (Neon) - Serverless 아키텍처를 도입하여 인프라 관리 부담을 줄이고, 관계형 데이터베이스의 무결성을 바탕으로 복잡한 교대 근무 제약 조건 데이터를 유연하게 제어
